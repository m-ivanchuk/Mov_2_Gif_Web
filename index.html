✿ About Kawaii Web Converter ✿

Kawaii Web Converter is a private, offline-first, serverless video-to-GIF converter. It allows you to convert files without uploading them anywhere, preserving your privacy and data completely.

How It Works

This tool uses FFmpeg WebAssembly (Wasm), a port of the world's most powerful open-source video processing suite compiled to run natively in your browser. All conversion tasks run inside your local browser sandbox, using your own device's CPU/GPU. No servers are involved!

Conversion Setting Guidelines

  • Dithering (Palette Matching):
    • Floyd-Steinberg (Default): Best for videos with gradients (sky, sunsets, photography). It creates the smoothest color transitions.
    • Bayer Matrix: Gives an ordered dot pattern like retro pixel art. Very cool retro aesthetic and often results in smaller files for solid colors.
    • Sierra Lite: A faster, lighter dither with low noise. Good balance of speed and quality.
    • Heckbert Color: Advanced dynamic color quantization, producing a rich color space.
    • No Dither: Best for cartoon/pixel-art styles without color gradients. It keeps files extremely small.
  • Colors:
    • 256 colors: Max quality, richest look, but largest file size.
    • 64-128 colors: Sweet spot for web animations and fast loading.
    • 16-32 colors: Retro/lo-fi look, ultra-light files.
  • FPS (Frames Per Second):
    • Standard GIFs use 10-15 FPS. High values like 30 or 60 FPS look smooth but make GIF file sizes massive because GIFs do not have modern video compression.
  • Scale:
    • Scaling down (e.g., 50% or 75%) is the easiest way to significantly reduce the output GIF file size.

We respect your privacy. Here is exactly what happens when you use Kawaii Web Converter:

1. Zero Server Uploads

Your files are never uploaded to any server. When you drop a video file, it is loaded directly into your browser's local memory sandbox. The entire conversion is executed locally by your browser using WebAssembly.

2. Security of 30MB Downloads

When you first load the site, it downloads about 30 megabytes of static assets. These assets include the WebAssembly core files (the compiled version of the FFmpeg engine).

Why this is safe:

  • These are static files loaded from GitHub Pages, served securely over HTTPS.
